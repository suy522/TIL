2. pre-commit(자동 고침) 사용법
   1_ pip install pre-commit.   -인스톨
   2_ pre-commit --v   -버전확인
   3_ touch .pre-commit-config.yaml  -파일 만들기
   4_ pre-commit sample-config > .pre-commit-config.yaml  -샘플넣기
   5_ cat .pre-commit-config.yaml   -확인
   6_ pre-commit run  -시동
   7_ vi로 작업하기
   8_ 작업한 파일 add로 넣기
   9- pre-commit run 으로 잘못된곳 오토수정
   10- 파일 수정했으니 add로 다시 추가 이후 commit하고 push




3. git 기본구동

 1_홈페이지에서 새로운 프로젝트 만들기. --- > 코드복사.    -프로젝트가져올때는 dev경로에서 가져오기
 2_ git clone 복사한코드                     -넣어서 끌어오기
   2_1  gitignore 사용
 3_ 폴더와 파일 만들기 mkdir(폴더) , touch   (파일)
 4_ vi 를 활용하여 안에 코드나 내용작성       -i로 작성 esc로 나오기 :wq로 저장
 5_ 작성한 파일을 git add  로 넣기.           -파일을 수정할경우 다시 add해줘야함
 6_ add 한 파일을 git commit으로 저장하고 뜨는 vi에 변경한 내용 적기
 7_ git push origin main 으로 홈페이지에 적용




4. branch

 분기점 전환으로 다른 곳에서 작업후 끌어오기 가능

1.분기점 확인 git branch
2.분기점 만들기 git branch 이름
3.분기점으로 이동하기. git switch 이름
4.분기점에서 vi를 활용하여 작업하기   -commit단계까지 해도됨 push까지 가능하나 추천하지 않음
5.다시 main 으로 돌아와서 분기점에서 작업한 내역 가져오기 git merge 이름
6.잘 가져왔나 확인 vi,cat 등등
7.분기점 삭제 git branch -D 이름
8.push 로 홈페이지 적용
